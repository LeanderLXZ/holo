# 关键决策 —— 压缩版 ADR

持久性工程决策的压缩日志。每条决策一个条目：一行决策正文 +
一行理据 + 一个指向权威来源（代码路径、文档段、日志文件）的指针。

冗长的讨论链与每条决策背后的推理过程存放于
`logs/change_logs/` —— 本文件是索引，不是讨论本身。

## 格式

```
N. <一行决策陈述>。
   <一行理据>。
   → <指向权威来源的指针>
```

条目仅追加不修改，按落地顺序编号。
**不要重新编号** —— 下游引用（代码注释、change log、
其他 ai_context 文件中）都引用编号。当后续决策取代
前面的决策时，新增条目，并在旧条目处追加一行
"superseded by #N" 注记；但不要删除旧条目。

## 段（按主题组织）

随决策日志增长，挑选稳定的主题化标题 —— 例如
"Data Separation"、"Runtime Loading"、"Schema Bounds"。
同一章节内的决策仍按全局（整文件）编号。
